ManagedStackExplorer

Managed Stack Explorer provides call stack for .NET applications using managed code with thread local variables. The API as of now does not provide values for the local variables. When it is available we can update it. It is a single executable without any other dll's. It uses Costura to embed dll.

This is built using Microsoft.Diagnostics.Runtime

The debug shim works specific to processor and would not be able to get callstacks if it is not. So x86 exe cannot get call-stacks of x64. That's reason for x86 and x64 specific exe's. There is no difference in code other than how it is compiled.
